# 🚀 Scallex

O **Scallex** é uma landing page moderna desenvolvida para apresentação de serviços digitais, com foco em clareza de comunicação, experiência do usuário e estrutura orientada à conversão.

🔗 **Acesse o projeto:**  
https://scallex.vercel.app/

---

## 🧠 Visão geral

O projeto tem como objetivo central a construção de uma interface web eficiente para exposição de serviços digitais, permitindo apresentar propostas de valor de forma direta, organizada e estratégica.

A aplicação segue princípios modernos de desenvolvimento front-end, priorizando legibilidade de código, reutilização de componentes, separação de responsabilidades e uma experiência consistente entre diferentes dispositivos.

---

## 🎯 Objetivo

Desenvolver uma landing page capaz de apresentar serviços digitais de forma estruturada, guiar o usuário através de um fluxo lógico de informação, destacar problemas e soluções de forma clara e direcionar o usuário para ações específicas (CTA).

---

## 🏗️ Arquitetura e estrutura

O projeto foi organizado com base em componentização, facilitando manutenção e escalabilidade da aplicação.

A estrutura principal inclui:

- Header  
- Hero  
- Problems  
- Solution  
- Comparison  
- CaseStudy  
- FAQ  
- CTA  
- Footer  

Além dos arquivos principais:

- `App.tsx`  
- `main.tsx`  
- `index.css`  

---

## ⚙️ Desenvolvimento

Durante o desenvolvimento, foram aplicados conceitos fundamentais de front-end moderno.

A interface foi construída com componentes reutilizáveis, garantindo organização modular e facilidade de manutenção. O layout foi desenvolvido com abordagem responsiva (mobile-first), assegurando adaptação adequada em diferentes dispositivos.

Também foram considerados aspectos de usabilidade, como hierarquia visual, espaçamento consistente e clareza na navegação.

Elementos interativos, como botões de ação (CTA) e integrações de contato, foram implementados para melhorar a experiência do usuário e direcionar interações.

---

## 🧱 Estrutura da landing page

A aplicação segue um fluxo lógico de conversão, organizado nas seguintes seções:

- **Hero** — apresentação inicial  
- **Problems** — identificação de dores  
- **Solution** — proposta de solução  
- **Comparison** — diferenciação  
- **Case Study** — contextualização  
- **FAQ** — redução de objeções  
- **CTA** — direcionamento para ação
